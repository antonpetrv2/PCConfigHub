PCConfigHub REST API

API Documentation

These routes power the Next.js web client and the future Expo mobile app. All API handlers return JSON except this documentation page.

Basics Schemas Auth Parts Uploads Configurations Comments Admin Errors

Basics

Base URL in local development: http://localhost:3000

Protected endpoints accept the browser cookie auth_token or Authorization: Bearer <jwt>.

Successful and failed JSON responses use the same envelope:

{
  "data": {},
  "error": null,
  "meta": null
}

Public parts/configurations submitted by regular users become visible to everyone only after approval. Private records are visible to their owner immediately.

Schemas

Common enums

visibility: "private" | "public"
role: "user" | "moderator" | "admin"
category: "motherboard" | "cpu" | "gpu" | "ram" | "psu" | "case" | "storage" | "soundcard"

Part base payload

{
  "name": "Ryzen 7 7800X3D",
  "manufacturer": "AMD",
  "model": "100-100000910WOF",
  "description": "Optional text",
  "visibility": "public",
  "category": "cpu",
  "specs": {}
}

Auth

POST /api/auth/register

Creates a user account and returns user data. Password must be at least 8 characters.

{ "name": "Alex Builder", "email": "alex@example.com", "password": "password123" }

POST /api/auth/login

Authenticates and sets the auth_token cookie. Also returns auth data for API clients.

{ "email": "alex@example.com", "password": "password123" }

GET /api/auth/me

Returns the current user or 401 when unauthenticated.

POST /api/auth/logout

Clears the auth_token cookie and returns { "success": true }. Bearer-token clients should also clear their locally stored token.

Parts

GET /api/parts

Lists approved public parts plus the current user's own parts.

Query params: category, search, page, limit.

GET /api/parts/[id]

Returns one accessible part with specs and image list.

POST /api/parts

Requires auth. Accepts multipart form data with payload JSON. Image can be passed as image or uploaded separately with imageUrl in payload.

FormData:
payload = {
  "name": "NZXT H5 Flow",
  "visibility": "public",
  "category": "case",
  "specs": {
    "formFactor": ["atx", "micro-atx"],
    "psuFormFactor": "atx",
    "maxGpuLength": 365
  },
  "imageUrl": "https://..."
}

PUT /api/parts/[id]

Requires ownership. Updates base fields, specs, visibility and optionally image.

Send imageUrl: null to remove the current image, omit it to keep the current image.

DELETE /api/parts/[id]

Requires ownership. Soft deletes the part and removes stored images from R2 when possible.

Uploads

POST /api/upload

Requires auth. Accepts multipart form data field file.

Image files are resized to fit within 1600x1200, converted to WebP and uploaded to Cloudflare R2. Non-image files are uploaded as-is.

curl -X POST -F "file=@case.jpg" http://localhost:3000/api/upload
{ "data": { "url": "https://pub-...r2.dev/parts/case-ab12.webp" }, "error": null, "meta": null }

Configurations

GET /api/configs

Lists approved public configurations plus the current user's own configurations.

Query params: page, limit.

GET /api/configs/[id]

Returns configuration details, parts, cover image and compatibility result.

POST /api/configs

Requires auth. Creates a configuration. A case part is required and incompatible part sets are rejected.

{
  "name": "Quiet 1440p build",
  "description": "Optional text",
  "visibility": "private",
  "parts": [1, 2, 3, 4]
}

PUT /api/configs/[id]

Requires ownership. Updates name, description, visibility and selected parts.

DELETE /api/configs/[id]

Requires ownership. Soft deletes the configuration.

POST /api/configs/check-compatibility

Checks compatibility before a configuration exists. Useful for builder previews.

{ "parts": [1, 2, 3] }

POST /api/configs/[id]/check-compatibility

Checks compatibility in the context of an existing configuration id.

Comments

GET /api/parts/[id]/comments

Lists approved comments for a public part.

POST /api/parts/[id]/comments

Requires auth. Creates a pending comment.

{ "body": "Great value for this chipset." }

GET /api/configs/[id]/comments

Lists approved comments for a public configuration.

POST /api/configs/[id]/comments

Requires auth. Creates a pending comment.

Admin

Admin routes require admin. Approval routes also allow moderator where noted.

GET /api/admin/pending

Admin or moderator. Lists pending users/content/comments.

GET /api/admin/users

Admin only. Lists users.

PUT /api/admin/users/[id]/role

Admin only. Body may be { "role": "moderator" } or the raw role string.

POST /api/admin/parts/[id]/approve

Admin or moderator. Approves a pending public part.

POST /api/admin/parts/[id]/reject

Admin or moderator. Optional body: { "reason": "..." }.

POST /api/admin/configs/[id]/approve

Admin or moderator. Approves a pending public configuration.

POST /api/admin/configs/[id]/reject

Admin or moderator. Optional body: { "reason": "..." }.

POST /api/admin/comments/[id]/approve

Admin or moderator. Approves a pending comment.

POST /api/admin/comments/[id]/reject

Admin or moderator. Optional body: { "reason": "..." }.

404Record not found.
422Validation or compatibility error. Validation details are returned in meta.details.
500Server configuration or unexpected failure.
